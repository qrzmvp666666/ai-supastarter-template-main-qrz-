## Why Next.js App Router?

The App Router introduced in Next.js 13 is a game changer. It embraces React Server Components by default, which means less JavaScript shipped to the browser and faster page loads.

## Setting Up Your First Project

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

## Key Concepts

### Server Components

By default, all components in the `app/` directory are **Server Components**. They run on the server, can fetch data directly, and never ship their code to the client.

```tsx
// This runs on the server — no "use client" needed
export default async function Page() {
  const data = await fetch("https://api.example.com/data");
  const json = await data.json();

  return <div>{json.title}</div>;
}
```

### Client Components

Add `"use client"` only when you need interactivity or browser APIs:

```tsx
"use client";

import { useState } from "react";

export function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}
```
