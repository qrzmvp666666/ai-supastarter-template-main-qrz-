## Warum Next.js App Router?

Der mit Next.js 13 eingeführte App Router ist ein Wendepunkt. Er setzt standardmäßig auf React Server Components, was bedeutet: weniger JavaScript im Browser und schnellere Ladezeiten.

## Dein erstes Projekt aufsetzen

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

## Kernkonzepte

### Server Components

Alle Komponenten im `app/`-Verzeichnis sind standardmäßig **Server Components**. Sie laufen auf dem Server, können direkt Daten abrufen und schicken keinen Code an den Client.

```tsx
export default async function Page() {
  const data = await fetch("https://api.example.com/data");
  const json = await data.json();

  return <div>{json.title}</div>;
}
```

### Client Components

Füge `"use client"` nur hinzu, wenn du Interaktivität oder Browser-APIs benötigst:

```tsx
"use client";

import { useState } from "react";

export function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}
```
