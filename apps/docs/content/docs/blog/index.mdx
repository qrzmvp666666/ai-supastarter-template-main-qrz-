## 概述

博客系统位于 `apps/marketing`，使用 `@content-collections/core` + `@content-collections/mdx` 处理 MDX 文章，无需数据库。所有文章以 `.mdx` 文件形式存放，构建时静态生成页面。

## 所需工具

| 工具 / 服务 | 本地开发 | 生产环境 | 说明 |
|------------|:--------:|:--------:|------|
| Node.js ≥ 20 | ✅ | ✅ | 构建 MDX 内容 |
| 图片托管服务 | — | ✅ | 封面图推荐存放到公开 CDN 或 public/ 目录 |
| 无需额外服务 | ✅ | ✅ | 完全静态，无数据库依赖 |

## 文章目录

```
apps/marketing/content/posts/
├── my-first-post.mdx          # 默认语言（英文）
├── my-first-post.fr.mdx       # 法语版本
└── my-first-post.zh.mdx       # 中文版本
```

## 文章 Frontmatter 字段

```mdx
---
title: "文章标题"
date: 2024-01-01
image: "/images/blog/my-post.jpg"
authorName: "作者姓名"
authorImage: "/images/authors/avatar.jpg"
authorLink: "https://twitter.com/author"
excerpt: "文章摘要，显示在文章列表中。"
tags:
  - 技术
  - 产品
published: true
---

这里是文章正文（MDX 格式）。
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | ✅ | 文章标题 |
| `date` | date | ✅ | 发布日期，格式 `YYYY-MM-DD` |
| `image` | string | ✅ | 封面图路径或 URL |
| `authorName` | string | ✅ | 作者名 |
| `authorImage` | string | — | 作者头像路径 |
| `authorLink` | string | — | 作者主页链接 |
| `excerpt` | string | — | 列表摘要（不填则截取正文） |
| `tags` | string[] | — | 文章标签，支持多个 |
| `published` | boolean | — | `false` 则不在列表显示，默认 `true` |

## 添加文章

1. 在 `apps/marketing/content/posts/` 创建 `.mdx` 文件
2. 填写 frontmatter 字段
3. 编写正文内容（支持 MDX 组件）
4. 本地运行 `pnpm --filter marketing dev` 即时预览

## 多语言支持

文件名格式：`<slug>.<locale>.mdx`

```
my-post.mdx       → 默认语言路由 /blog/my-post
my-post.zh.mdx    → /zh/blog/my-post
my-post.fr.mdx    → /fr/blog/my-post
```

语言配置在 `packages/i18n/config.ts` 中管理。

## Content Collections 配置

Schema 定义位于 `apps/marketing/content-collections.ts`，如需新增字段，在此文件的 `defineCollection` 中添加。
