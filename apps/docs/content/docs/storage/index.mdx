## 概述

项目使用 S3 兼容协议进行文件存储（用户头像、组织 Logo 等），配置通过环境变量注入，存储逻辑位于 `packages/storage`。

## 所需工具

| 工具 / 服务 | 本地开发 | 生产环境 | 说明 |
|------------|:--------:|:--------:|------|
| Docker（MinIO） | ✅ | — | 本地 S3 兼容存储，随 `docker compose up` 自动启动 |
| AWS S3 | — | ✅ | 标准对象存储 |
| Cloudflare R2 | — | ✅ | 无出口流量费，推荐生产使用 |
| DigitalOcean Spaces | — | ✅ | 简单易用 |
| Supabase Storage | — | ✅ | 配合 Supabase 数据库使用 |
| `avatars` bucket | ✅ | ✅ | 本地自动创建，生产需手动创建并设置公开读 |

## 本地开发（MinIO）

Docker 启动时自动完成，直接填入以下变量：

```dotenv
S3_ACCESS_KEY_ID="minioadmin"
S3_SECRET_ACCESS_KEY="minioadmin"
S3_ENDPOINT="http://localhost:9000"
S3_REGION="us-east-1"
NEXT_PUBLIC_AVATARS_BUCKET_NAME="avatars"
```

MinIO 控制台：[http://localhost:9001](http://localhost:9001)（账号 `minioadmin` / `minioadmin`）

## 生产环境配置

### Cloudflare R2（推荐）

```dotenv
S3_ACCESS_KEY_ID="your-r2-access-key"
S3_SECRET_ACCESS_KEY="your-r2-secret-key"
S3_ENDPOINT="https://<account-id>.r2.cloudflarestorage.com"
S3_REGION="auto"
NEXT_PUBLIC_AVATARS_BUCKET_NAME="avatars"
```

### AWS S3

```dotenv
S3_ACCESS_KEY_ID="AKIA..."
S3_SECRET_ACCESS_KEY="your-secret"
S3_ENDPOINT="https://s3.amazonaws.com"
S3_REGION="us-east-1"
NEXT_PUBLIC_AVATARS_BUCKET_NAME="avatars"
```

### DigitalOcean Spaces

```dotenv
S3_ACCESS_KEY_ID="your-spaces-key"
S3_SECRET_ACCESS_KEY="your-spaces-secret"
S3_ENDPOINT="https://<region>.digitaloceanspaces.com"
S3_REGION="<region>"
NEXT_PUBLIC_AVATARS_BUCKET_NAME="avatars"
```

## 生产 Bucket 设置

1. 在存储控制台创建名为 `avatars` 的 bucket
2. 设置**公开读**策略（用于头像图片展示）
3. 配置 CORS，允许来自 SaaS 应用域名的请求
